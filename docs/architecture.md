# Architecture Overview

Worldsmith Sandbox separates its procedural generation logic from its graphical user interface to ensure high performance and maintainability.

## Core Map Engine (`AzgaarEngine`)
At the heart of Worldsmith is the `AzgaarEngine`, located in `python_fmg/core/azgaar_engine.py`. This procedural map generation engine is responsible for:

1. **Voronoi Tessellation**: Generating an initial random point distribution and triangulating it using SciPy's `Voronoi` to create the discrete map cells.
2. **Heightmap Pipeline**: Constructing tectonic plate boundaries and iterating cellular automata to form organic landmasses and mountain ranges.
3. **Hydrology Routing**: Tracing downhill slopes from elevated terrain to carve realistic river paths that flow toward the ocean.
4. **Cultural Propagation**: Simulating the expansion of states and religions across valid biomes.

## User Interface (`WorldsmithMainWindow`)
The UI is written entirely in PyQt6 (`python_fmg/main.py`). It consists of two main layouts:

- **The Control Panel (Left)**: Contains sliders for map parameters, a Markdown-enabled notebook for writing lore, and the AI terminal interface.
- **The Map Canvas (Right)**: The `MapViewerWidget` (`python_fmg/renderers/map_viewer.py`) physically renders the Voronoi mesh generated by the engine. It handles all matrix transformations (panning/zooming) and captures mouse events for the magic brushes.

## The Lore Database
Data persistence for the world's lore is managed via a local SQLite database (`lore_forge_world.db`). When the user writes a note in the notebook, it is saved directly into the database.

## AI Workers (`OllamaPromptWorker`)
Heavy analytical LLM requests (for timeline auditing and lore organization) are completely detached from the Qt Event Loop. Found in `python_fmg/core/ai_worker.py`, these inherit from `QThread` to issue HTTP requests to the local Ollama server in the background, emitting Qt Signals when the analytical response stream is ready.
